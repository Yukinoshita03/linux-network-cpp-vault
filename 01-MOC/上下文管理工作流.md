# 上下文管理工作流

## 目标

用 Obsidian + GitHub 把学习、项目、面试准备沉淀成长期知识库，减少每次和 AI 或自己复盘时重复解释背景的问题。

## 每次学习后记录

每个知识点至少写清楚：

1. 这个东西解决什么问题。
2. 它处在协议栈哪一层。
3. 关键机制是什么。
4. 能不能用命令、抓包或代码验证。
5. 面试时怎么讲。

## 和 AI 协作时怎么用

向 AI 提问时，优先贴相关笔记链接或摘录这几块内容：

- 当前目标
- 已完成实验
- 报错或现象
- 相关命令输出
- 希望产出的形式

## GitHub 提交节奏

建议每天至少一次提交：

```bash
git status
git add .
git commit -m "Update Linux networking notes"
git push
```

提交信息可以按内容写：

```text
Add ARP notes
Add mini-router topology
Update BGP state machine notes
Summarize epoll ET mode
```
