## 使用方式

每天只做三件事：

1. 补一个知识点。
2. 写一段能运行的代码或实验。
3. 记录一个可复述的面试答案。

不要只收藏资料。这个方向的竞争力来自能解释真实报文路径、能写转发代码、能用抓包和 Linux 工具定位问题。

## Agent 联动

这个库已经兼容 llm-wiki 风格的 Agent 工作流。后续让 AI/Agent 处理本方向问题时，优先让它读取：

1. `index.md`
2. `purpose.md`
3. `.wiki-schema.md`
4. 对应主题笔记

新素材放入 `raw/`，沉淀后的知识放入 Obsidian 主目录或 `wiki/`。
