# 研究目的与方向

## 核心目标

建立一个可被人和 Agent 共同维护的长期知识库，用来系统掌握 Linux 网络协议栈、通信协议栈、转发面、控制面协议、C/C++ 工程能力和求职算法基础，并最终服务于相关岗位求职和项目作品沉淀。

## 关键问题

1. Linux 中一个报文从网卡进入到被转发出去的完整路径是什么？
2. 转发面工程师需要掌握哪些数据结构、协议和性能优化手段？
3. 如何实现一个能跑、能抓包验证、能写进简历的 mini-router？
4. 如何用 C++ 实现简化 BGP/OSPF 状态机，并和 FRR 对接？
5. 面试时如何把 Linux、协议、C/C++、项目经验串成可信叙述？
6. 如何建立可迁移的算法题型框架，并在 C/C++ 笔试和面试中稳定实现？

## 研究范围

**涵盖：**

- Linux 系统编程
- socket / epoll / raw socket
- TCP/IP、Ethernet、ARP、IPv4、ICMP
- Linux 内核网络栈
- Netfilter、TC、XDP/eBPF
- DPDK 基础
- LPM、Trie、转发表
- OSPF、BGP、FRR
- C/C++ 网络工程能力
- C/C++ 求职算法与数据结构、高频题型复盘
- 项目设计、实验记录、面试答案

**暂不涵盖：**

- Linux 桌面使用
- 前端开发
- 普通 Java 后端
- 缺乏题型总结、错误复盘和求职价值的机械题海记录

## Agent 协作目标

后续 Agent 应把这个库当作业务背景知识源：

- 回答前先查已有笔记。
- 修改方案要符合当前学习路线和项目目标。
- 新知识优先沉淀到 Obsidian。
- 不把一次性聊天当成长期记忆。
- 重要变更后提交并推送 GitHub。
