## 核心洞见

1. 指针算法不能只记代码，要先确定循环不变量：二分查找维护“目标仍可能位于哪个闭区间”，快慢指针维护“已整理区间里有什么”，滑动窗口维护“当前窗口和是多少”。
2. 每轮循环必须保证状态向结束条件推进。二分查找要移动 `left/right`，快慢指针要移动 `fast`，滑动窗口缩小时要移动 `slow`；遗漏推进通常会造成死循环或重复计算。
3. 记录答案的变量和维护窗口状态的变量职责不同。`result` 只记录最优长度，`sum` 才表示窗口元素和；缩小窗口时应修改 `sum`，不能误改 `result`。
4. 双指针覆盖原数组前必须确认不会破坏尚未读取的数据。`removeElement` 可以安全覆盖，因为 `slow <= fast`；`sortedSquares` 从后写回原数组时却可能覆盖右端未读取元素，因此线性解通常需要结果数组。
5. 复杂度最优不等于必须原地。`sortedSquares` 使用额外数组可以做到 `O(n)`；若坚持直接修改原数组，简单方案是先平方再排序，时间退化为 `O(n log n)`。
6. 二分边界题的关键不是“遇到相等就返回”，而是决定相等时继续向左还是向右：`< target` 找第一个 `>= target`，`<= target` 找第一个 `> target`。
7. `searchInsert` 的答案天然是第一个 `>= target` 的位置；`searchRange` 可以用“第一个 `>= target`”和“第一个 `> target`”相减得到首尾范围。

## 题型一：二分查找

### 循环不变量

采用左闭右闭区间 `[left, right]`：

- 初始化：`left = 0`，`right = nums.size() - 1`。
- 循环条件：`left <= right`，因为 `left == right` 时仍有一个候选元素。
- `nums[mid] < target`：舍弃 `[left, mid]`，令 `left = mid + 1`。
- `nums[mid] > target`：舍弃 `[mid, right]`，令 `right = mid - 1`。
- 区间为空仍未命中时返回 `-1`。

```cpp
int search(vector<int>& nums, int target) {
    int left = 0;
    int right = static_cast<int>(nums.size()) - 1;

    while (left <= right) {
        int mid = left + (right - left) / 2;

        if (nums[mid] < target) {
            left = mid + 1;
        } else if (nums[mid] > target) {
            right = mid - 1;
        } else {
            return mid;
        }
    }

    return -1;
}
```

### 我的易错点

- 把 `vector` 长度写成 `nums.len`，正确接口是 `nums.size()`。
- 使用 `left < right`，漏查最后一个候选元素。
- 查找失败时返回 `mid`，语义不正确，应返回 `-1`。
- `return` 语句漏写分号。

## 题型二：搜索插入位置

这道题不需要专门讨论“找到了没有”。无论目标值是否存在，答案都是数组中第一个 `>= target` 的位置：

- 目标存在时，这是目标值的第一个位置。
- 目标不存在时，这是保持有序所需的插入位置。
- 如果目标大于所有元素，答案可以等于 `nums.size()`，表示插入到数组末尾。

```cpp
int searchInsert(vector<int>& nums, int target) {
    int left = 0;
    int right = static_cast<int>(nums.size()) - 1;

    while (left <= right) {
        int mid = left + (right - left) / 2;

        if (nums[mid] < target) {
            left = mid + 1;
        } else {
            right = mid - 1;
        }
    }

    return left;
}
```

循环结束时，`right` 左侧的元素都小于 `target`，`left` 右侧的元素都不小于 `target`，因此 `left` 就是插入位置。这也是手写版的 `lower_bound`。

### 我的易错点

- `mid` 必须包含左边界偏移量，不能写成 `right - left + 1`。
- 相等时不能立即返回，因为题目要的是第一个 `>= target` 的位置。
- `left` 可能等于数组长度，访问 `nums[left]` 前必须先判断 `left < nums.size()`。

## 题型三：查找元素的首尾位置

这道题可以拆成两个边界问题：

1. 找第一个 `>= target` 的位置，也就是左边界。
2. 找第一个 `> target` 的位置，减一后就是右边界。

```cpp
vector<int> searchRange(vector<int>& nums, int target) {
    int length = static_cast<int>(nums.size());

    int left = 0;
    int right = length - 1;

    while (left <= right) {
        int mid = left + (right - left) / 2;

        if (nums[mid] < target) {
            left = mid + 1;
        } else {
            right = mid - 1;
        }
    }

    int leftBound = left;

    if (leftBound == length || nums[leftBound] != target) {
        return {-1, -1};
    }

    left = 0;
    right = length - 1;

    while (left <= right) {
        int mid = left + (right - left) / 2;

        if (nums[mid] <= target) {
            left = mid + 1;
        } else {
            right = mid - 1;
        }
    }

    int rightBound = left - 1;
    return {leftBound, rightBound};
}
```

例如 `nums = [5, 7, 7, 8, 8, 10]`、`target = 8`：

- 第一个 `>= 8` 的位置是 `3`。
- 第一个 `> 8` 的位置是 `5`，所以最后一个 `8` 是 `5 - 1 = 4`。
- 返回 `{3, 4}`。

### 我的错误复盘

- `mid = right - left + 1` 没有加上 `left`，而且没有右移，第一次循环就可能访问越界。
- 相等时令 `left = mid + 1` 找到的是第一个 `> target` 的位置，即 `upper_bound`，不是左边界。
- `rightBound` 不应从 `0` 开始线性扫描；用第二次边界二分可以把整体复杂度保持为 `O(log n)`。
- 空数组或目标值大于全部元素时，左边界可能等于 `length`，不能直接访问 `nums[leftBound]`。

## 题型四：移除元素

### 循环不变量

`[0, slow)` 始终保存已经筛选出的、所有不等于 `val` 的元素。`fast` 负责读取，只有读到有效元素时才写到 `slow` 并推进 `slow`。

```cpp
int removeElement(vector<int>& nums, int val) {
    int length = static_cast<int>(nums.size());
    int slow = 0;

    for (int fast = 0; fast < length; ++fast) {
        if (nums[fast] != val) {
            nums[slow] = nums[fast];
            ++slow;
        }
    }

    return slow;
}
```

原先“先写入、再判断”的版本结果仍然正确，因为无效值最终会被后续有效元素覆盖；但它会产生不必要的写操作，也不如“判断后写入”直观。

## 题型五：有序数组的平方

原数组有序，但平方后最大值可能来自最左端的负数，也可能来自最右端的正数。因此比较两端平方值，把较大的值从结果数组末尾向前放。

```cpp
vector<int> sortedSquares(vector<int>& nums) {
    int length = static_cast<int>(nums.size());
    int left = 0;
    int right = length - 1;
    int write = right;
    vector<int> result(length);

    while (left <= right) {
        int leftSquare = nums[left] * nums[left];
        int rightSquare = nums[right] * nums[right];

        if (leftSquare > rightSquare) {
            result[write--] = leftSquare;
            ++left;
        } else {
            result[write--] = rightSquare;
            --right;
        }
    }

    return result;
}
```

### 为什么不能简单地线性原地覆盖

例如 `[-4, -1, 0, 3, 10]`：先放入 `100` 后，如果下一步把 `16` 写到原数组倒数第二个位置，就会覆盖尚未读取的 `3`。所以常规双指针解法使用额外结果数组。

- 标准解：时间 `O(n)`，额外空间 `O(n)`。
- 简单原地解：全部平方后调用 `sort`，时间 `O(n log n)`。
- 理论上的线性原地归并实现复杂，不适合作为这道题的常规答案。

## 题型六：长度最小的子数组

这道题利用数组元素均为正数：右端加入元素时窗口和不会减小，左端移除元素时窗口和不会增大。因此可以用滑动窗口在线性时间内寻找最短长度。

```cpp
int minSubArrayLen(int target, vector<int>& nums) {
    int length = static_cast<int>(nums.size());
    int slow = 0;
    int sum = 0;
    int result = length + 1;

    for (int fast = 0; fast < length; ++fast) {
        sum += nums[fast];

        while (sum >= target) {
            result = min(result, fast - slow + 1);
            sum -= nums[slow];
            ++slow;
        }
    }

    return result == length + 1 ? 0 : result;
}
```

### 我的易错点

- `int fast, slow = 0;` 只初始化了 `slow`，`fast` 仍是未初始化值。
- 缩小窗口时误写成 `result -= nums[slow]`，正确变量是 `sum`。
- 只有刷新最短长度时才移动 `slow`，可能让循环状态不再变化并形成死循环。
- 手动控制外层循环时漏掉 `fast++`，导致同一个元素被重复加入窗口。
- 使用 `slow < fast` 会漏掉长度为 `1` 的有效窗口。

## 统一检查清单

写完指针类算法后，按顺序检查：

1. 每个指针是否初始化？
2. 当前维护的是闭区间还是半开区间？
3. `left == right` 或窗口长度为 `1` 时是否仍会检查？
4. 每条循环路径是否至少推进一个指针或改变退出条件？
5. 写回原数组是否会覆盖尚未读取的数据？
6. 答案变量、窗口状态变量和指针变量是否混用？
7. 边界二分结束后，`left` 是否可能等于 `nums.size()`？
8. 无解时的返回值是否符合题目约定？
