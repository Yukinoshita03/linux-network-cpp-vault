# STL 迭代器入门：本质、区间与算法关系

## 先给一句话定义

迭代器可以先理解成：

- **泛化过的指针**

它的核心作用是：

- 给标准库一个**统一的遍历接口**
- 让算法不用关心底层容器到底是 `vector`、`list` 还是 `map`

所以 STL 的思路不是：

- “每个容器都配一套自己专属的算法”

而是：

- “容器负责存数据”
- “迭代器负责把数据暴露成统一遍历接口”
- “算法只认迭代器区间”

---

## 为什么 STL 需要迭代器，而不是直接用下标

因为不是所有容器都有下标。

比如：

- `vector` 有 `v[i]`
- `deque` 也能随机访问
- 但 `list` 没法高效 `l[i]`
- `map` / `set` 也不是按整数下标访问

如果标准库算法都要求：

- “给我一个下标”

那很多容器根本接不进去。

所以 STL 才设计成：

- 不统一“访问方式”
- 统一“遍历方式”

而这个统一遍历方式，就是迭代器。

---

## 最基础的几个动作

先看最常见写法：

```cpp
std::vector<int> v = {10, 20, 30};

auto it = v.begin();  // 指向第一个元素
std::cout << *it;     // 10

++it;                 // 往后走一个
std::cout << *it;     // 20
```

你可以先把它类比成指针：

- `begin()`：起点
- `end()`：终点后一位
- `*it`：取当前元素
- `++it`：往后走

所以最基础的遍历就是：

```cpp
for (auto it = v.begin(); it != v.end(); ++it) {
    std::cout << *it << " ";
}
```

---

## 为什么 `end()` 不是最后一个元素

这是一个非常关键的点。

`end()` 指向的不是最后一个元素，
而是：

- **最后一个元素的后一个位置**

所以：

```cpp
v.begin()  // 第一个元素
v.end()    // 终点哨兵，不可解引用
```

不能写：

```cpp
*v.end();  // 错
```

这样设计的好处是，区间很好表达。

---

## STL 为什么总用 `[first, last)`

标准库算法最常见接口长这样：

```cpp
sort(first, last);
find(first, last, x);
count(first, last, x);
```

它们用的是：

- **左闭右开区间** `[first, last)`

意思是：

- 包含 `first`
- 不包含 `last`

这有几个好处：

1. 空区间很好表示：`first == last`
2. 长度很好算：对指针或随机访问迭代器是 `last - first`
3. 前一段的 `last` 可以直接作为后一段的 `first`

比如：

```cpp
[begin, mid)
[mid, end)
```

拼起来刚好是整个区间。

---

## 算法为什么都“围着迭代器转”

看两个例子：

```cpp
std::sort(v.begin(), v.end());
std::find(v.begin(), v.end(), 20);
```

算法根本不需要知道：

- `v` 是 `vector`
- 里面怎么存
- 容量多大

它只需要知道：

- 从哪里开始
- 到哪里结束
- 怎么往后走
- 怎么取当前元素

这就是迭代器最核心的价值：

- **把“容器差异”隔离掉**

所以 STL 的本质其实很漂亮：

- 容器和算法是解耦的
- 中间靠迭代器粘起来

---

## 不同容器的迭代器，能力不一样

这也是迭代器最容易被忽略的一点。

### `vector` 的迭代器

能力强，支持随机访问。

比如：

```cpp
it + 3
it - 2
it1 < it2
```

### `list` 的迭代器

能力弱一些，只能一步一步走。

比如：

```cpp
++it;
--it;
```

但不能默认写：

```cpp
it + 3;  // 不行
```

### `map` / `set` 的迭代器

也不是随机访问迭代器。

它们支持往前往后走，但不支持拿“位置差”当数组那样玩。

所以你要记住：

- **不是所有迭代器都像数组指针一样强**

这就是为什么有些算法能用于 `vector`，却不能用于 `list`。

---

## 一个最重要的例子：为什么 `sort` 不是所有容器都能用

```cpp
std::vector<int> v = {3, 1, 2};
std::sort(v.begin(), v.end());   // 可以
```

但：

```cpp
std::list<int> l = {3, 1, 2};
std::sort(l.begin(), l.end());   // 不行
```

因为 `std::sort` 需要：

- **随机访问迭代器**

而 `list` 只有双向迭代器，不够强。

`list` 要排序，通常得用它自己的成员函数：

```cpp
l.sort();
```

这个例子特别适合你建立直觉：

- 算法不是只看“元素类型”
- 还看“迭代器能力”

---

## 常见迭代器类别，先记够用线

你不用一上来背标准定义，先记这个强弱层次就够：

1. 输入迭代器：能往前读
2. 前向迭代器：能多次往前走
3. 双向迭代器：能前后走
4. 随机访问迭代器：能跳着走，像数组

你现在最该记住的是容器对应关系：

- `vector`：随机访问迭代器
- `deque`：随机访问迭代器
- `list`：双向迭代器
- `map` / `set`：双向迭代器
- `unordered_map` / `unordered_set`：前向迭代器

---

## 范围 `for` 和迭代器是什么关系

你平时经常写：

```cpp
for (auto& x : v) {
    std::cout << x << " ";
}
```

它本质上背后也是在用：

- `begin()`
- `end()`
- 迭代器递增

所以范围 `for` 只是：

- **更方便的迭代器语法糖**

日常遍历时它更舒服；
但一旦和 STL 算法配合，还是得回到迭代器区间。

---

## 你现在阶段最该掌握的 6 件事

1. 迭代器本质上是“统一遍历接口”，可以粗略理解成泛化指针
2. `begin()` 指向第一个元素，`end()` 指向尾后位置
3. STL 算法几乎都吃 `[first, last)` 这种迭代器区间
4. `*it` 是取值，`++it` 是往后走
5. 不同容器的迭代器能力不同，不是都支持 `it + n`
6. 算法是否能用，不只看容器，也看迭代器类别
