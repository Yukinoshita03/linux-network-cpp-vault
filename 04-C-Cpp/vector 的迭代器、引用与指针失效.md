# `vector` 的迭代器、引用与指针失效

## 为什么这页重要

`vector` 最常见、最危险的坑之一不是复杂度，而是：

- 你手里保存的迭代器
- 引用
- 指针

什么时候还有效，什么时候已经悬空。

## 最核心的根因

`vector` 的元素放在一整块连续内存里。  
一旦这块连续内存被重新分配，元素就会整体搬家。

所以：

- 原来指向旧内存的迭代器、引用、指针
- 都会失效

这就是一切规则背后的总原因。

## 先记一个大原则

### 不改容量的操作

通常只影响局部位置后的迭代器。

### 一旦扩容

通常全部失效：

- 所有迭代器
- 所有引用
- 所有指针

## 1. `push_back` / `emplace_back`

### 情况 A：没有扩容

```cpp
std::vector<int> v;
v.reserve(10);
v.push_back(1);
auto p = &v[0];
v.push_back(2);   // 没扩容
```

这时：

- 指向原有元素的引用/指针/迭代器通常还有效
- 但 `end()` 会改变，所以旧的 `end()` 失效

### 情况 B：发生扩容

```cpp
std::vector<int> v = {1, 2, 3};
int* p = &v[0];
v.push_back(4);   // 可能扩容
```

如果这次触发扩容：

- `p` 失效
- `v.begin()` 旧值失效
- `v.end()` 旧值失效
- 所有元素引用都失效

## 2. `insert`

插入比尾插更复杂，因为它还会移动插入点及之后的元素。

### 如果发生扩容

规则和前面一样：

- 全部失效

### 如果没发生扩容

那么：

- 插入位置之前的迭代器/引用/指针通常还有效
- 插入位置及其之后的通常失效

原因是：

- 后面的元素要整体后移
- 位置已经不是原来的那个对象布局了

## 3. `erase`

```cpp
auto it = v.begin() + 2;
v.erase(it);
```

`erase` 不会扩容，但会让后面的元素整体前移补洞。

所以：

- 被删位置及其之后的迭代器、引用、指针失效
- 之前的通常还有效

这也是为什么你不能删完以后还继续用原来的 `it`。

正确习惯是接住返回值：

```cpp
it = v.erase(it);
```

因为 `erase` 会返回“下一个有效位置”的迭代器。

## 4. `clear`

```cpp
v.clear();
```

这会销毁所有元素。

所以：

- 所有元素引用/指针/迭代器都失效

但通常：

- 容量还在
- 底层缓冲区不一定释放

所以“capacity 可能还在”不等于“旧元素地址还能用”。

## 5. `resize`

### 缩小

```cpp
v.resize(smaller);
```

- 被裁掉元素相关的引用/指针/迭代器失效
- 前面保留下来的元素通常还有效

### 变大

```cpp
v.resize(larger);
```

- 如果触发扩容，全部失效
- 如果没扩容，已有元素位置通常还稳定，但新元素被追加到尾部

## 6. `reserve`

这个很重要，因为很多人低估它的破坏性。

```cpp
v.reserve(100);
```

如果 `reserve` 真的导致重新分配：

- 所有迭代器
- 所有引用
- 所有指针

全部失效。

如果 `reserve` 请求不大于当前 capacity，没有重新分配，那就不会导致这些失效。

## 7. `shrink_to_fit`

如果库真的执行了缩容并重分配：

- 全部失效

但它只是请求，不保证一定执行。

## 8. 为什么“引用失效”比“值变了”更危险

失效的本质不是“值不对”，而是：

- 你拿着一个还像地址的东西
- 但它已经不再指向合法的当前元素

这就是典型悬空引用 / 悬空指针问题。

有时程序会立刻崩：

- 访问非法内存

有时更糟：

- 看起来还能跑
- 但读到的是脏数据

## 9. 一组最实用的经验法则

### 规则 1

只要操作可能触发扩容，就默认之前保存的迭代器/引用/指针全别信了。

### 规则 2

做 `erase` / `insert` 后，不要继续用旧位置附近的迭代器。

### 规则 3

如果你需要长期持有元素地址，`vector` 要非常小心。

### 规则 4

如果你已知后面会大量追加，先 `reserve()`，能减少失效次数。

## 10. 一句话总结

`vector` 的失效规则本质都来自“连续内存 + 可能整体搬家”；扩容时几乎全失效，不扩容时通常是局部位置之后失效。
