# C++ lambda 入门：本质、捕获与算法配合

## 先给一句话定义

lambda 可以先理解成：

- **现场写出来的匿名小函数**

它最常见的价值不是“语法更酷”，
而是：

- 你想把一小段行为立刻传给算法、回调、线程
- 但又不想专门跑去定义一个独立函数

所以它和 STL 算法库天然很搭。

---

## 为什么学完算法库，下一步自然就是 lambda

你前面已经见过这种东西：

```cpp
std::sort(v.begin(), v.end(), [](int a, int b) {
    return a > b;
});
```

这里第三个参数不是数据，
而是：

- “排序规则”

也就是说，算法除了要数据，
还常常要你提供：

- 比较规则
- 过滤规则
- 判断条件
- 回调逻辑

lambda 就是给这类“临时行为”服务的。

---

## 最基本长什么样

```cpp
[](int a, int b) {
    return a + b;
}
```

可以拆成 4 块：

1. `[]`：捕获列表
2. `(int a, int b)`：参数列表
3. `{ ... }`：函数体
4. 返回类型：通常编译器自动推导

所以你可以先把它类比成：

```cpp
int add(int a, int b) {
    return a + b;
}
```

只是 lambda 没有名字，而且可以直接写在使用现场。

---

## lambda 的返回值要不要声明

大多数时候：

- **不用你手写声明**

比如：

```cpp
auto f = [](int a, int b) {
    return a + b;
};
```

这里编译器会自动推导出返回类型是：

```cpp
int
```

所以你可以先把 lambda 的返回值规则记成：

1. 有 `return`，就根据 `return` 表达式推导
2. 没有 `return`，默认就是 `void`

比如：

```cpp
auto f1 = [] {
    std::cout << "hello\n";
}; // 返回 void

auto f2 = [](int x) {
    return x * 2;
}; // 返回 int
```

### 那什么时候要显式写

如果你想自己指定返回类型，
可以写成：

```cpp
auto f = [](int x) -> double {
    return x / 2.0;
};
```

这里的：

```cpp
-> double
```

就叫尾置返回类型。

### 什么时候显式写会更有必要

#### 1. 多个 `return` 不容易自动统一

比如：

```cpp
auto f = [](bool ok) {
    if (ok) return 1;
    return 2.0;
};
```

这类写法就容易出问题，
因为一个分支像 `int`，
一个分支像 `double`。

这时通常要自己写清楚：

```cpp
auto f = [](bool ok) -> double {
    if (ok) return 1;
    return 2.0;
};
```

#### 2. 你想返回引用

这是一个很容易忽略的点。

比如：

```cpp
int x = 10;
auto f = [&x]() -> int& {
    return x;
};
```

如果你真的想返回引用，
那显式写 `-> int&` 会更稳。

否则很多时候自动推导更像按值返回。

### 一句话收住

lambda 的返回值：

- **通常靠编译器自动推导**
- **只有在你想强行指定类型、统一多个分支、或返回引用时，才更常显式写 `-> type`**

---

## 最核心的不是参数，而是捕获

lambda 真正和普通函数拉开差距的地方是：

- **它可以顺手拿到外面的局部变量**

比如：

```cpp
int base = 10;

auto f = [base](int x) {
    return x + base;
};
```

这里 `base` 不是参数，
而是从外层拿进来的。

这就叫：

- **capture，捕获**

---

## 最常用的几种捕获写法

### `[]`

不捕获任何外部变量。

```cpp
auto f = [](int x) {
    return x * x;
};
```

### `[x]`

按值捕获 `x`。

```cpp
int x = 5;
auto f = [x]() {
    return x;
};
```

含义是：

- 把外面的 `x` 拷一份进 lambda 里

后面外面的 `x` 再变，
lambda 里面这份默认不跟着变。

### `[&x]`

按引用捕获 `x`。

```cpp
int x = 5;
auto f = [&x]() {
    x++;
};
```

含义是：

- lambda 里直接操作外面那个 `x`

### `[=]`

把用到的外部变量都按值捕获。

### `[&]`

把用到的外部变量都按引用捕获。

---

## 值捕获和引用捕获，到底差在哪

这是 lambda 最重要的一层。

### 一个先纠正的说法

`[base]` 和 `[&base]`：

- **不是在传参**
- 而是在决定 lambda 怎么捕获外层变量 `base`

更准确地说：

- `[base]`：把 `base` 的当前值拷一份进 lambda
- `[&base]`：让 lambda 里直接引用外面的那个 `base`

所以它不是：

- “一个传进去不变值，一个传应用”

而更像：

- “一个拿副本，一个连原件”

### 值捕获

你可以理解成：

- **拿一份副本进来**

所以更像“快照”。

### 引用捕获

你可以理解成：

- **直接连到外面那个变量**

所以更像“别名”。

这和前面你学过的拷贝 / 引用直觉是连起来的。

---

## 一个最典型的例子

```cpp
int x = 10;

auto f1 = [x]() {
    std::cout << x << "\n";
};

auto f2 = [&x]() {
    std::cout << x << "\n";
};

x = 20;

f1();  // 10
f2();  // 20
```

因为：

- `f1` 里是值捕获，拿到的是旧副本
- `f2` 里是引用捕获，连着外面的真变量

你也可以直接把这组关系记成：

- `[base]` ≈ 拷贝一份 `base`
- `[&base]` ≈ 给 `base` 起了个别名

---

## `mutable` 是干嘛的

默认情况下，
值捕获进来的那份副本在 lambda 里不能随便改。

比如：

```cpp
int x = 10;
auto f = [x]() mutable {
    x++;
    std::cout << x << "\n";
};
```

`mutable` 的意思不是：

- 把外面的 `x` 改了

而是：

- 允许你修改“lambda 里面那份值捕获副本”

所以它改的是副本，不是外面原变量。

---

## lambda 最常见的使用场景

### 1. 配合算法库

```cpp
std::sort(v.begin(), v.end(), [](int a, int b) {
    return a > b;
});
```

```cpp
v.erase(std::remove_if(v.begin(), v.end(), [](int x) {
    return x % 2 == 0;
}), v.end());
```

### 2. 作为条件判断器

```cpp
auto it = std::find_if(v.begin(), v.end(), [](int x) {
    return x > 100;
});
```

### 3. 做回调

比如以后你学线程、异步、GUI、网络库时，经常会把一小段逻辑作为回调传进去。

---

## 它的本质其实不是“函数”，而是“函数对象”

严格一点说，
lambda 背后会生成一个匿名类型，
这个类型实现了：

- `operator()`

所以它本质更像：

- **编译器帮你偷偷生成的一个小函数对象**

这也是为什么它能“带着捕获的数据一起走”。

你现阶段不用抠展开细节，
只要记住一句：

- lambda 不是魔法，它本质是带状态的可调用对象

---

## 新手最容易踩的 3 个坑

### 1. 引用捕获悬空

```cpp
auto make_lambda() {
    int x = 10;
    return [&x]() {
        return x;
    };
}
```

这是危险的。

因为函数返回后：

- 外面的 `x` 已经死了
- lambda 里还拿着它的引用

这就是悬空引用。

### 2. 以为值捕获会跟着外面同步变化

不会。

值捕获拿到的是当时那一份副本。

### 3. 把 `[&]` 滥用成默认习惯

`[&]` 很方便，
但也更容易把生命周期问题带进来。

所以写长期保存、异步执行的 lambda 时，
要特别警惕引用捕获。

---

## 这一页真正该记住的 6 句话

1. lambda 是现场写出来的匿名小函数
2. 它最常用来把“规则”或“行为”传给算法和回调
3. `[]` 里写的是捕获规则，不是参数
4. `[x]` 是值捕获，`[&x]` 是引用捕获
5. 值捕获更像副本，引用捕获更像别名
6. lambda 本质上是编译器生成的函数对象

---

## 面试够用线

至少要能顺着讲清：

1. lambda 是匿名可调用对象，常用于算法、回调、临时规则
2. 它的核心语法由捕获列表、参数列表、函数体组成
3. 值捕获是复制一份进来，引用捕获是直接绑定外部变量
4. `sort` / `remove_if` / `find_if` 经常和 lambda 连用
5. 引用捕获如果跨作用域存活，可能出现悬空引用
